第四周-Webpack学习1

前端 Webpack
发布日期:   2020-07-28

简述

webpack是一个现代JavaScript应用程序的静态模块打包器。

理解

打包对象

  • 如果不进行配置，webpack只认识JavaScript

打包文件

  • 如果你的项目需要多个js文件就会有以下问题
    • 存在非常严重的依赖顺序问题，逻辑多，文件多，项目的复杂度提高
  • webpack打包就是来解决上述问题的

打包技能——loader

  • 高阶语法（ES6），衍生出的第三方语法（typescript）
  • 这些都是浏览器（或者低版本浏览器）无法理解的
  • webpack的loader可以解决上述问题

打包技能——plugin

  • 打包文件时加一些小动作
  • 可以随时去除

模块化

引入

  • 想要使用独立、封装的对象
  • JavaScript作用域有全局和局部（函数）

创建模块

  • 利用函数作用域和立即执行函数

    (function(window){
    var name='tyt';
    var sex='male';
    function tell(){
        console.log(name,sex);
    }
    window.myModule={tell};
})(window)
    • 内部使用闭包保存变量
      • 同时变为私有属性（外部无法访问）
    • 接受window参数，并将想要暴露的东西挂载出去
    • 立即执行函数保证了作用域的封装

优点

  • 作用域封装
  • 接触耦合
  • 重用性

历史

  • nodejs开启了js全栈大门
    • AMD（异步模块定义） 规范
    • Commonjs || CMD 规范
      • 两者相似
      • commonjs是js在后端语言的规范（nodejs）：模块、文件操作、操作系统底层
      • CMD 仅仅是模块定义
    • UMD 通用模块定义，一种既能兼容 AMD 也兼容 commonjs 还能兼容浏览器环境的万能代码
    • ES6 规范（最推荐的方法）
  • 集中包管理的方式
    • bower 只能下载前端的js代码，在下载bootstrap的时候回自动地下载jQuery
    • browserify 解决require可以运行在浏览器，组装代码
    • webpack 打包工具 主流

webpack的打包利器

loader

  • 标准js引入过程中，只能引入js文件
  • webpack利用loader可以使其引入其他文件，例如
    • css
    • png
    • scss
    • …
  • webpack本身并不包含loader，需要按需下载
  • 可以看做是一个文件加载器

使用

  • 下载

    npm install xxx-laoder -S

  • 配置（webpack版本不同，配置方式不同）

plugin

  • 监听某些事件，在必要时刻发挥作用 例如：
    • uglifyjs-webpack-plugin可以将打包好的文件进行压缩（去除注释，空格…）

使用

  • 下载

    npm install uglifyjs-webpack-plugin -S

  • 配置（webpack版本不同，配置方式不同）
