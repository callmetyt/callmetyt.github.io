第六周-typescript基础学习3

前端 typescript
发布日期:   2020-08-14

类

基础

  • 声明

    class Point {
  x: number;
  y: number;
  constructor(x: number, y: number) {
    this.x = x;
    this.y = y;
  }
  getPosition(): void {
    console.log(`(${this.x},${this.y})`);
  }
}
    • 基本语法和ES6一致
    • 声明变量的时候当然要定义数据类型

  • 使用

    const point = new Point(1, 2);
point.getPosition();
//(1,2)

  • 继承和ES6的语法一致

访问修饰符

对类的属性或方法的访问特性进行修饰

和c++的属性修饰符差不多一样

  • public

    • 公共属性
    • 默认修饰符

  • private

    • 私有属性
    • 只能在类的定义中进行访问和设置
    • 继承的类也无法访问和设置

  • protected

    • 受保护属性
    • 和private差不多·一样
      • 只是可以被子类访问

  • 注意点

    • 类的constructor也可以被修饰符限定
      • 如果不是public，那么无法通过new生成新的实例
      • JS中new的本质是调用了构造函数

readonly

  • 限定一个属性或方法为只读
    • 无法被实例修改
    • 只能在类的定义中初始化或者constructor中修改
  • 独立于public protected private

参数属性

  • class A{
    constructor(public haha:number){}
}
const a = new A(1);
console.log(a.haha);
//1
    • 在构造函数constructor中对参数使用访问修饰符，可以直接定义实例上
    • 只能用在构造函数上
    • 必须有访问修饰符，不然实例不会有这个属性

静态属性

  • 和ES6语言一致
  • 通过static修饰的属性或者方法只能通过类名访问

可选类属性

  • 使用?:即可，和之前的函数的可选参数一致
  • 可以用在属性声明上，说明这个属性不是必须的

取值器

  • 和ES6的语法一致

  • settter getter

    class A{
    constructot(){}
    get a(){
        return this.a;
    }
    set a(v:any){
        this.a=v;
    }
}

抽象类

  • 声明

    abstract class People {
  constructor(public name: string) {}
  abstract printName(): void;
}
    • 这是一个抽象类
    • 里面有一个抽象函数
    • 抽象类无法被实例化，抽象函数不能直接书写具体实现

  • 使用

    class Man extends People {
  constructor(name: string) {
    super(name);
  }
  printName() {
    console.log(this.name);
  }
}

const test2 = new Man("haha");
test2.printName();
//haha
    • 通过继承抽象类获取接口
    • 抽象函数需要自己实现
      • 抽象函数只能存在于抽象类中
      • 抽象函数不能标记返回值

  • Ts2.0版本之后，可以定义抽象属性和存取器

    • 效果和抽象函数一样，只能存在于抽象类中，必须由子代实现

实例类型

  • 声明实例类型

    class People {
  constructor(public x: number) {}
}

const test: People = new People(4);

    • 这里的变量的类型声明可以去掉

    • 如果另一个类的构造函数和People一样，则可以：

      class OtherPeople{
  constructor(public x: number) {}
}

const test: People = new OtherPeople(4);

类接口

  • 声明接口

    interface test{
    name:string
}

  • 类使用接口

    class Haha implements test {
  constructor(public name: string) {}
}
    • 注意使用的语法class Haha implements test

  • 注意点

    • 接口保证的是实例，而不是类本身
      • 静态属性实现接口无效

接口继承类

  • 声明接口并继承类

    class A {
  protected a: string = "hjja";
  constructor(a: string) {
    this.a = a;
  }
}

interface I extends A {}

  • 使用接口定义类

    class B extends A implements I {
  constructor(a: string) {
    super(a);
  }
  getA(): void {
    console.log(this.a);
  }
}

const b = new B("ss");
b.getA();
//ss
    • B要实现接口I但是A的属性是受保护的，无法直接访问
    • 类B就要继承类A，才能实现接口I
      • 即：class B extends A implements I

泛型使用类

  • 类的类型作为参数的写法

    class Infos {
  public age: number = 18;
}
const create = <T>(c: new () => T): T => {
  return new c();
};
console.log(myCreate(Infos));
//输出一个Infos类
    • 注意create函数的参数写法：c: new() => T
      • 接受一个类标识符，c返回一个类
      • 最后再返回一个实例

枚举

数字枚举

  • 声明

    enum Status {
  test1,
  test2,
  test3,
}
console.log(Status.test1, Status["test2"]);
//0 1
    • 默认索引从0开始
    • 可以自定义索引（参见基础数据类型中的枚举）

  • 注意点

    const getId = () => 2;
enum Status {
  test1,
  test2 = getId(),
  test3 = 3,
}
    • test3必须指定初始索引，因为此处使用了计算属性

反向映射

  • 枚举对象内部实质上是属性和索引之间的映射，他们互为键和值

字符串枚举

  • 声明

    enum Status {
  test1 = "dfs",
  test2 = "sf",
}

console.log(Status.test1, Status["test2"]);
//dfs sf
    • 注意字符串枚举不存在反向映射

异构枚举

  • 同时存在数字枚举和字符串枚举会构成异构枚举,其中数字成员会形成反向映射
  • 不建议使用

枚举成员类型和联合枚举类型

  • 枚举成员

    enum Status {
  on,
  off,
}

interface I {
  haha: Status.on;
}

const test: I = {
  haha: 1,
};

  • 联合枚举

    enum Status {
  on,
  off,
}

interface I {
  haha: Status;
}

const test: I = {
  haha: Status.on,
};

运行时的枚举

  • 枚举类型是存在真正的代码的，可以再运行时代码中使用（settimeout等等）
  • 接口之类的代码并不真正存在（编译为JavaScript之后），不能再运行时代码中使用

const enum

  • const修饰的枚举会变成c语言当中的#define x x，变成“宏定义”
  • const枚举不存在真正的代码，只是在编译阶段进行值的替换

类型推论和兼容性

类型推论

基础

  • typescript会自行推论变量可能的类型

    let name = 'tyt';
//name推断为string类型，赋给其他类型会报错

多类型联合

  • let arr = [1, "s", true];

    • 此时arr会自动推断为含有数值、字符串和布尔类型的数组，即

      let arr: (number | string | boolean)[];

上下文类型

  • 例如事件函数处理的第一个参数是event事件对象，这是上下文决定好的类型

类型兼容性

基础

  • 例子

    interface Info {
  name: string;
}
let infos: Info;
const infos1 = {
  name: "tyt",
};
const infos2 = {
  age: 17,
};
const infos3 = {
  name: "xxx",
  age: 18,
};
infos = infos1;//不会报错
infos = infos2;//报错
infos = infos3;//不会报错
    • 属性多了无所谓，少了不行****
    • 存在递归判断
      • 对象嵌套对象，属性也要完全一致

函数兼容性

参数个数
  • let x = (a: number) => a;
let y = (a: number, b: number) => a;
x = y;//报错
y = x;//不报错
    • 接受的函数的参数只能少，不能多
      • 原因是某些参数可能用不到
参数类型
  • 只要参数类型不同，就不能相互赋值
返回值类型
  • 只要返回值类型不同，就不能互相赋值
    • 多个类型兼容少类型
    • 返回的值可以不同
可选参数和剩余参数

  • 可选参数在判定兼容性的时候，应视为本身类型和undefined类型的结合

  • 剩余参数可以和任意数量的同类型的参数兼容

    let x = (...args: string[]): string => {
  return "1";
};
let y = (a: string, b: string): string => {
  return "haha";
};
x = y;//不报错
y = x;//不报错
    • 不会报错
参数双向协变

  • 例子

    let x = (arg: string): string => {
  return "1";
};
let y = (a: string | number): string => {
  return "haha";
};
x = y;//不报错
y = x;//报错
    • x的参数类型只能是string，y的参数类型还可以是number，不兼容
    • 可以通过tsconfig.json配置进行设置，从而两个都不报错
函数重载

  • 例子

    function merge(a: number, b: number): number;
function merge(a: string, b: string): string;
function merge(a: any, b: any): any {
  return a + b;
}

function sum(a: number, b: number): number;
function sum(a: any, b: any): any {
  return a + b;
}

let func = merge;
func = sum;//报错
    • 此时函数merge的重载比函数sum多了一种，不兼容

枚举

  • 数字枚举兼容数字
    • 不同的数字枚举之间不兼容

类

  • 兼容性比较

    • 只比较实例属性
    • 不比较方法和静态属性

  • 例子

    class Animal {
  public static age: number;
  constructor(public name: string) {}
}

class People {
  public static age: string;
  constructor(public name: string) {}
  getss() {}
}

class Food {
  constructor(public name: number) {}
}

let animal: Animal;
let people: People = new Animal("ss");//不报错
let food: Food;

animal = people;//不报错
animal = food;//报错
    • animal和people的静态属性不同，但是实例属性相同，不报错；food实例属性不同，报错
      • 实例方法也不会影响

  • 继承问题

    class P {
  private name: string;
  constructor(name: string) {
    this.name = name;
  }
}

class C extends P {
  constructor(name: string) {
    super(name);
  }
}

class T {
  private name: string;
  constructor(name: string) {
    this.name = name;
  }
}

let c: C = new C("ss");
let t: C = new T("ss");//报错
    • 即使成员一样，private和protected修饰的属性独一无二，必须来自同一个类，否则不兼容

泛型

interface Data<T> {
  data: T;
}

let data1: Data<number> = {
  data: 1,
};
let data2: Data<string> = {
  data: "str",
};

data1 = data2;//报错
