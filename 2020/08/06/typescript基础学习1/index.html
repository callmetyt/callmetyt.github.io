typescript基础学习1

前端 typescript
发布日期:   2020-08-06

！语法转换！

https://www.typescriptlang.org/play/index.html

这里可以书写typescript语法，网站会自动转换为JavaScript语法

基础数据类型

定义数据

let test:boolean=true;
  • 必须在变量后面声明数据类型

简单数据类型

  • 布尔类型 boolean

  • 数值类型 number

    • 支持2（0b）、8（0o）、16（0x）进制

  • 字符串类型 string

    • 支持ES6的模板字符串

  • 数组类型

    • let arr:number[]或者let arr:Array<number>

      arr是一个可以存放数值类型的数组

    • let arr:(string|number)[]

      arr是一个可以存放数值和字符串的数组

  • any类型 any

    • any类型可以存放任何数据（能不用就不用）

  • void类型 void

    • let v:void

      v只接受 undefined 和 null

      • 赋值 null 需要在 tsconfig.json中修改配置

        即："strictNullChecks": true,这一项设置为false，与此同时，其他基本数据类型可以赋值 undefined 或 null

  • null和undefined

    • let u:undefined let n:null

      u 只接受 undefined

      n 只接受 null

  • never 类型

    • 不存在的数据类型
    • 定义变量是无意义的，主要用于无返回值函数
      • 抛出错误的函数
      • 死循环函数

  • object 类型

    • let obj:object={
    name:'haha'
}

      对象类型

复杂数据类型

  • 元组类型

    • let tuple;[string,number]

      tuple是一个长度固定为2，第一个元素是字符串，第二个是数值的数组

    • 声明方式必须是变量名:[数据类型...]

  • 枚举类型

    • enum Test{//声明枚举类型一般变量名首字母大写
    a,
    b,
    c
}

    • 打印输出Test会是

      0:'a',1:'b',2:'c'
a:0,b:1,c:2

      可见枚举类型的特性，数据和索引一一对应

    • 可以这样声明

      enum Test{
    a=1,
    b,
    c=2
}

    • 结果会是 a:1 b:2 c:2

      • 初始化赋值会影响后续索引
      • 索引可以重复

类型断言（转化）

  • 变量可以存在多种数据类型

    • let test:string|number

      test可以同时接受字符串和数值

  • 可以认为转化数据的类型

    function test(haha: string | number): void {
  console.log(typeof (<string>haha));
  console.log(typeof (haha as number));
}
    • 上述两种方法可以修改变量的数据类型，前提是变量存在多种数据类型形式
    • 修改数据类型只是为了防止使用相应的方法时Ts报错，因为Js是弱语言，没有显式类型转换的代码

接口

声明

  • 定义接口

    interface NameInfo {
  first: string;
  last: string;
}
    • NameInfo的对象有两个属性
    • string类型的first和string类型的last

  • 使用

    let obj:NameInfo={
    first:'tyt',
    last;'lyx'
}
    • 此时对象obj只能有接口定义的属性，数据类型必须一致

属性检查

  • 可选属性

    interface Vegetable{
    color?:string,//定义方式为 ?:
    type:string
}
    • 此时定义的Vegetable接口汇中的color属性可有可无

  • 多余属性

    • 方法一 强制定义

      interface Vegetable{
    color:string,
    type:string
}

let obj:Vegetable={
    color:'red',
      type:'apple',
    price:'15'
} as Vegetable
      • 最后的as Vegetable会强制声明成功

    • 方法二 索引签名

      interface Vegetable{
    color:string,
    type:string,
    [prop:string]:any
}
      • 此时Vegetable可以拥有无限制数量的属性（只要有前两个定义好的属性）

    • 方法三 类型兼容器

      const b=a
/*
只要b里面的属性，a里面都有就行，多了无所谓
*/

  • 只读属性

    interface Test{
    readonly type:string
}
    • 此时Test接口的type属性只能被读取，不能修改

函数接口

  • 声明

    interface Func{
    (num1:number,num2:number):number
}
    • 两个number类型的参数
    • 返回一个number类型值
    • 定义的参数名不必相同，但是类型必须一致

  • 使用

    const add:Func=(n1,n2)=>n1+n2
    • 此时函数接口限定了add函数的形式

索引签名

  • 声明

    interface Test{
    [id:number]:'number'
}
    • 属性名为number类型，值为number类型
    • 索引签名只能是string或者number

  • 使用

    let test:Test={
    1:12,
    2:55
    //...数量不限
}

继承

  • 属性可以继承

    interface A{
    color:string,
    name:string
}

interface B extends A{
    special:string
}
    • 此时B接口拥有A接口的两个属性（color和name）

混合对象类型

  • 闭包函数的Ts特性实现

    interface Counter{
    ():void,
    count:number
}
    • 接口含有一个返回值为void的函数和一个number类型的count属性
    let getCounter = (): Counter => {
  const c = function (): void {
    c.count++;
  };
  c.count = 0;
  return c;
};
    • 声明返回类型为Counter
    • 返回了一个函数，函数有一个number类型的count属性
    let counter=getCounter();
counter();
console.log(counter.count);//1
counter();
console.log(counter.count);//2
    • 达到闭包的效果
