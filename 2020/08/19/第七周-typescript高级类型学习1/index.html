第七周-typescript高级类型学习1

前端 typescript
发布日期:   2020-08-19

交叉类型

  • 多个数据类型交叉（都要有）

  • 关键运算符&

  • 声明且使用

    const mergeFunc = <T, U>(arg1: T, arg2: U): T & U => {
  let res = <T & U>{};
  res = Object.assign(arg1, arg2);
  return res;
};

console.log(mergeFunc({ a: "a" }, { b: "b" }));
//{a: "a", b: "b"}
    • 函数返回T和U的交叉类型（两个必须都有）
    • 声明变量使用类型断言（<>或者as）
    • 普通数据类型使用&运算符会生成never数据类型

联合类型

  • 声明多个数据类型，只有一个就行
    • 关键运算符|
    • let a:string | number变量a可以是字符串或者是数值

类型保护

  • 问题引入

    let getRandom = () => {
  let arr:(number | string)[] = [123, "abc"];
  if (Math.random() > 0.5) {
    return arr[0];
  } else {
    return arr[1];
  }
};

let res = getRandom();
if (res.length) {//报错
  console.log(res.length);//报错
} else {
  console.log(res.toFixed());//报错
}
    • 函数返回值是随机的，number没有length，string没有Fixed，Ts无法判断数据类型，会报错

  • 解决1：函数保护

    function isString(val: any): val is string {
  return typeof val === "string";
}
    • 注意返回值：val is string，此处说明变量val是一个string
    • 同时Ts判断出res若不是string就是number，else条件语句也不会出错
    • 适用于所有JS数据类型的保护

  • 解决2：typeof保护（推荐）

    if (typeof res === "string") {
  console.log(res.length);
} else {
  console.log(res.toFixed());
}
    • JS中typeof仅是类型判断语句，Ts中却可以进行类型保护
    • 效果和函数保护一样
    • 短小精悍，易于维护

  • 解决3：instanceof

    class test1 {
  public name: string = "a";
  constructor() {}
}

class test2 {
  public age: number = 18;
  constructor() {}
}

let getRandomClass = () => {
  return Math.random() > 0.5 ? new test1() : new test2();
};

let res = getRandomClass();

if (res instanceof test1) {
  console.log(res.name);
} else {
  console.log(res.age);
}
    • 和typeof一样，instanceof在Ts中赋予了类型保护的效果
    • instanceof适用于判断类的类型

null和undefined深入

  • undefined可以是未定义类型的子类型

    let val = 123;
//此时val相当于 number | undefined
    • 可以在配置文件中修改
    • 可选参数实际上也和未定义类型一样

类型断言（null）

  • 如果一个变量断言不是null，可以这么书写

    let test: number | null;
function haha(a: number | null) {
  test = a;
  console.log(test.toFixed());//报错
}
    • test可能是null，就没有toFixed方法
    • 解决方法
      • test!.toFixed()
      • test?.toFixed()
      • 两种方法都指明变量不会是null

类型别名

  • 基础

    type myString = string;
let test: myString;
    • 和let test:string效果一样

  • 泛型

    type Test<T> = {
  x: T;
  y: T;
};

let haha: Test<number> = {
  x: 2,
  y: 1,
};
    • 和接口十分类似（推荐使用接口）

  • 注意点

    • 不能使用 extends implements
    • 和接口可以相互类型兼容
    • 推荐使用接口
