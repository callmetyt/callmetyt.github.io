Vue基础

无标签
发布日期:   2020-06-25

准备开始

历史

angular 2009年

react 2013年

vue 2014年，(尤雨溪)中国人创建，现在美籍

框架和库

库(小而精)

  • jQuery库 -> DOM操作+请求
  • art-template -> 模板引擎

框架(大而全，可能包含多个库)

  • DOM操作+请求+模板引擎+路由功能

代码上

  • 库是引用 js 文件，调用函数，自己把控库的代码
  • 框架在帮我们运行我们写好的代码
    • 初始化自身的行为
    • 执行你编写的代码
    • 释放一些资源

      安装

直接官网下载 js 文件

推荐：使用 npm 进行下载(有效配合其他模块使用)

模板语法

基础

<div id="app">
    <h2>
        {{msg}}    //模板标记
    </h2>
</div>
//script
new Vue({
    el:'#app', //绑定目标
    data:{    //数据属性(正确用法在指令中)
        msg:'haha'
    },
    template:`<div>{{msg}}</div>`    //直接定义渲染目标，这将会跳过原html中书写的，不去渲染
})

模板标记支持变量运算，DOM操作,比如

{{1+1}}    //渲染为2
{{str.split('').reserve().join('')}}    //会将数据属性中的str变量翻转
{{1>2?'真':'假'}}    //渲染为假

注意只能包含单个表达式

Vue对象

new的基础上可以返回一个Vue对象

内部拥有我们定义的数据

内部早就定义好的属性会以$开头

指令

简介

  • vue提供了一些对于页面+数据更为方便的输出，这些操作叫做指令，以 v-xxx表示
  • 指令封装了一些DOM行为
  • 注意，指令进行操作时内部的值均为data数据，如果使用原生形态，内部则是字符串

v-text v-html

data:function(){    //其实这里应该要用函数返回值，后期讲解为什么，以后就用函数形式
    return{
        msg:'指令操作'
    }
},
template:`
    <h2>{{msg}}</h2>
    <p v-text='msg'></p> //相当于innertext
    <div v-html='msg'></div> //相当于innerhtml
`

注意 Mustache 语法(双大括号)，会把其中的内容解释为普通文本，如果是 html 代码，请使用 v-html之类的

v-if v-else v-show

data:function(){
    return{
        msg:true
    }
},
template:`
    <p v-if='msg'>if显示</p>    //字符串内的值为真，则显示
    <p v-else>else显示</p>    //为假显示else
    <p v-if='Math.random()>0.5'>随机显示</p> //值必须是字符串，可以嵌套函数
    <p v-show='msg'>show显示</p>    //show显示，和if差不多
`

if和show的区别

v-if 实际上是对DOM中的 appendChild() 和 removeChild() 的操作，v-show 是对 css 的 display 属性的操作，所以 v-if 开销较大，不适合频繁切换的组件

v-if-else

字面意思，可以多重嵌套 if

key管理可复用元素

Vue 会尽可能高效地渲染元素，通常会复用已有元素而不是从头开始渲染

也就是说，相同的元素标签，Vue不会重新渲染，只是替换变化的值

使用 key 属性标志需要区别的元素，即可重新渲染

v-bind

data:function(){
    return{
        msg:'hahaah'
    }
},
template:`
    <div v-bind:class='msg'></div> //渲染后div会得到一个 hahaah 的class
    <div :class='msg'></div> //v-bind的简写形式
`

v-on

template:`
    <div v-on:事件名='调用函数'></div> //绑定事件函数
    <div @:事件名='调用函数'></div> //v-on的简写
`
methods:{
    待调用函数名(){}
}

注意：当调用函数重新渲染时，是整个模板重新渲染

v-for

data:{
    lists:[
        {id:1,name:'hah'},
        {id:2,name:'ladjf'}
    ],
    obj:{
        name:'tyt',
        age:'19'
    }
},
template:`
    <ul>
        <li v-for='item in lists'> //遍历数组
            <h5 v-html='item.id'></h5>
            <h5 v-html='item.name'></h5>
        </li>
    </ul>    //一个ul列表，并且遍历data中的lists，然后输出数据到视图上

    <ul>    //遍历对象
        <li v-for='(value.key) in person'> //注意value遍历person，key则是索引
            {{key}}={{value}}
        </li>
    </ul>
`

v-model 双向数据绑定

templat:`
    <input type="text" v-model="msg">
    <h3 v-html="msg"></h3>
`,
data(){
    return{
        msg:'hahaha'
    }
}    //当input的值变化时，h3的值也会变化

v-model 实际上是一种语法糖，可以用以下组合实现

v-value="xxx" v-on:input=function(e){this.msg=e.target.value}
//注意this指向实例化的Vue对象，e.target指向触发事件的元素

数据传递(基础)

父组件 → 子组件

//1.在使用子组件时绑定属性
data(){return{msg:'xxx'}},
template:`
    <Vson :childData='msg'/>
`
//2.在子组件内部props绑定属性
props:['childData']
//3.在子组件中使用
<p v-html='childData'></p>

注意绑定属性后在子组件内部使用绑定名

子组件 → 父组件

//1.父组件在子组件绑定自定义事件
<div @childHandler='change'></div>
methods:{
    change(){}
}
//2.子组件中触发子组件自定义事件
<div @click='changeFather'></div>
methods:{
    changeFather(){
        this.$emit('childHandler')
    }
}
//3.$emit 第一个参数是父组件自定义的事件名，可以再传入参数当做父组件事件调用函数的参数
//4.自定义事件一定通过 $emit 触发
